„das universum“

... is a website mainly for younger users, such as attendees of primary schools, but also for everybody who is interested in the universe. As a brand, we want to stand out by creating content that is easily understandable as well as visually pleasing for children. This website was created in order to establish a more firmer brand identity. Desgined in the style of the main website of 'das universum', it presents a set of guidelines for when you are referring to our brand.

Naming conventions

The name 'das universum' actually speaks for itself. It is written in lower case because it makes our brand stand out. Children often don't care about capitalization. “das universum” must always be written in lower case. Do not capitalize „das“ or „universum“ or both. Do not link the two words together.

Fonts

The main heading of the homepage and any other headings use the font Astrolyte. Astrolyte is a font that represents the spirit of our brand. It conveys the feeling of the galaxy and the universe.

The simpler font Roboto is used for plain text, for example in the biography of each planet. This is because our focus lies on imparting knowledge. A more thrilling font would be distracting and it would be harder for the user to focus on the text, especially for children.

Our color palette

“zodiac purple” is used for headings and buttons

“cosmic blue” is used for links and with input types

plain text is either written in "noir" or "soft blanc". Since the background of the whole website is rather dark, we chose "soft blanc" for background fields.

Logo

The Logo of “das universum“ is a spiral that symbolizes the milky way. The brand uses this symbol in order to represent the universe with everything it includes, like planets and stars for example. Do not change the logo in any way. When using:

do not put the text inside the spiral

do not put the text below the spiral

CSS Methodology

Rules

Indentation

  • use tabs instead of spaces
  • always use semicolons
  • avoid px, whenever it's possible
  • use em and % instead

Folder structure

  • App folder:
    • html files
    • Images folder
    • JavaScript folder
    • SCSS folder
  • Doc folder
  • Bin folder

In order to avoid confusion, every type of code (e.g. css, html, js) has it's own folder.

Since we use SASS as a preprocessor, all scss files start with underline (see example folder structure)

Class naming convention

  • if you want to assign a class name which consists of two words use camelCase
  • if the class name contains a verb and a noun then write the verb before the noun

Component Library

Headings

Headings are styled in small-caps and in the color zodiac purple. However, you can decide whether you want to capitalize the words or not. They can be written next each other or with a space in-between. Do not style headings in lower case only.

Lists

For lists we use unordered lists with stars instead of dots. This is a referral to the topic of our website. Also, it makes things more interesting for children. Do not use ordered lists with enumerations. Do not use dots instead of stars.

Links

Links are not underlined and the style after visitation is the same as before. When hovering over the link it turns into italic and changes to the colour "cosmic blue". The cursor turns into a pointer.

Elements

All text-fields and buttons have rounded corners. The border radius can be chosen freely, as long as it is not too round. Buttons have a shadow in the color "soft blanc" below and on the right side. This gives the impression that they are floating through the galaxy and makes it easier for children to understand that they can click on them.

Radio Buttons, which are used for the quiz, have a custom design. They are bigger than normal radio buttons. When checked, they don't show a circle inside but a checkmark instead. To indicate the hover, when deciding which one to click, the check field turns "cosmic blue". This also happens when a radio button is checked.

Media Blocks:

For pictures, always use the best quality that is possible. When displaying a picture gallery give the pictures a thin border in "soft blanc" in order to make them stand out from dark the background.

Pictures of planets have a transparent background. They also have a personality (represented by a "face") to make them more interesting for children. Personalizing the planets gives you a higher chance of children remembering them.

Navigation

For the main menu navigation use a simple burger menu instead of a full list of links that is always present. This makes the homepage seem lass cramped. To help the user navigate easily through the website, use a bread-crumb navigation. Users will always know exactly on which page they currently are and can trace their steps back to the homepage.
